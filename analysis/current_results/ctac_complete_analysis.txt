TABLE 1: V2V PDR (GRIDLOCK, 352 VEHICLES)
================================================================================

Distance Filter: <= 300m
  This filter was inherited from the openCV2X codebase analysis scripts.
  It approximates the safety-critical communication range for C-V2X.
  3GPP TS 22.185 specifies minimum required communication ranges per
  use case (e.g., ~320m highway, ~150m urban), but does not prescribe
  a specific evaluation cutoff. 300m is a practical engineering choice.

  V2V data extends to 674m. Of total V2V receptions:
    Baseline: 5.40M within 300m out of 8.01M total (67%)
    CTAC K=2: 2.93M within 300m out of 4.37M total (67%)
    CTAC K=4: 2.87M within 300m out of 4.28M total (67%)

Calculation:
  1. For each (sender, receiver) link within 300m:
       link_PDR = packets_received / BSMs_generated_by_sender
  2. Per-vehicle PDR = mean(link_PDR) across all senders to that vehicle
  3. Overall PDR = total_received / total_expected across all links

TABLE 2: V2I PDR (GRIDLOCK, 352 VEHICLES)
================================================================================

Distance Filter: None (all distances included)
  V2I uses a single RSU (rsu[0]) at the intersection center.
  No distance cutoff is applied; packets are received or lost
  based on PHY-layer propagation and interference conditions.
  Max observed distance: 363.4m. Mean distance: ~170m.

Calculation:
  1. BSMs_generated(v) = sender_summary - ctacDeferred
  2. received_by_RSU(v) = count of packets from v received by rsu[0]
  3. PDR(v) = received_by_RSU(v) / BSMs_generated(v)
  4. Overall PDR = total_received_by_RSU / total_BSMs_generated

TABLE 4: MAC LAYER DROP ANALYSIS
================================================================================

Calculation:
  1. App BSMs Generated = sender_summary - ctacDeferred
  2. PHY Transmitted = tbSent:sum from .sca file
  3. Actual Drops = App BSMs Generated - PHY Transmitted
  4. Drop Rate = Actual Drops / App BSMs Generated

ACTUAL DROPS                             1,024             176             167
DROP RATE                                3.85%           1.31%           1.26%
--------------------------------------------------------------------------------
Drop Reduction vs Baseline                 ---          -82.8%          -83.7%

DCC Drops (packetDropDCC)                  677               0               0
Other MAC Drops                            347             176             167
Vehicles with Drops                        352             176             167

Note on drop signals:
  - packetDropDCC: Actual DCC-triggered drops (only confirmed drop signal)
  - grantBreak: SPS grant expirations, triggers reselection (NOT a drop)
  - missedTransmission: Missed TX slots, informational (NOT a drop)
  - droppedTimeout: Commented out in code, always 0

Drop pattern in CTAC scenarios:
  - 176/167 vehicles dropped EXACTLY 1 packet each
  - Likely caused by initial simulation startup transient
  - After transient, CTAC achieves near-perfect delivery
