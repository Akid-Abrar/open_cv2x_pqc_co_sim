SIGNALS EXPLAINED:
- grantBreak: Grant expires, triggers reselection (NOT a drop)
- missedTransmission: Missed transmission slots (NOT necessarily drops)
- packetDropDCC: ACTUAL packet drops by DCC
- droppedTimeout: Timeout drops (commented out = 0)

WHAT EACH NUMBER MEANS:
================================================================================

Generated:     App-layer BSMs created (sender_summary - ctacDeferred)
Transmitted:   PHY-layer TBs sent (tbSent:sum from .sca)
Dropped:       Packets lost between app and PHY
Drop%:         (Dropped / Generated) × 100
DCC Drops:     Confirmed drops by DCC (packetDropDCC:sum)

Other drops:   Total drops - DCC drops
               Baseline: 1,024 - 677 = 347 (other MAC layer issues)
               CTAC K=2: 176 - 0 = 176 (other MAC layer issues)
               CTAC K=4: 167 - 0 = 167 (other MAC layer issues)


IMPORTANT CLARIFICATIONS:
================================================================================

1. grantBreak (528/488/538) is NOT a drop counter
   → Tracks grant expirations that trigger reselection
   → Does NOT mean packets were dropped

2. missedTransmission is NOT a drop counter
   → Tracks missed transmission slots
   → Does NOT mean packets were dropped

3. ONLY packetDropDCC counts actual drops
   → Baseline: 677 DCC drops (confirmed)
   → CTAC: 0 DCC drops (DCC disabled)

4. Other drops (347/176/167) are from:
   → Queue overflow
   → MAC layer resource failures
   → Other untracked MAC issues
